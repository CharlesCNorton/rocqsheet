# Rocqsheet TODO

1. Round-trip `ls_merges` through the save format with `M=tl_col,tl_row,br_col,br_row` directive lines — currently merge regions vanish on save/load, so the user must rebuild them every session.
2. Round-trip `ls_charts` through the save format with `C=kind,tl_col,tl_row,br_col,br_row,title` directive lines — currently the `initial_charts` list at startup is the only chart layout and the user's edits to it are not persisted.
3. Round-trip `ls_edit_buf` through the save format with `E=col,row,raw_text` directive lines — currently a cell's underlying raw text (e.g. `=A1+B1` versus its evaluated value) is lost on load, so all formulas reload as their last-evaluated literal.
4. Add `V=<n>` as the first non-comment line of every save file, with a strict version validator at the top of `apply_load_lines` that refuses to load a future-version file and applies migrations for prior versions — currently the save format has no version field, so any future schema change silently corrupts old files.
5. Make `do_save` atomic by writing to `<path>.tmp`, `fsync`ing, and `rename`-ing into place — currently a crash mid-write leaves the user with a truncated corrupt file and no recovery.
6. Rotate `.bak` backups: `do_save` first renames `<path>` → `<path>.bak.1` and shifts older numbered backups, keeping the last three — currently the user has no recourse if a save corrupts.
7. Implement autosave: schedule a `do_save_as <path>.autosave` write every 30 seconds when `ls_dirty` is set — currently a crash between manual saves loses every edit since the last Ctrl+S.
8. Add crash-recovery on startup: if `<path>.autosave` exists and is newer than `<path>`, surface a modal at first frame asking the user to choose between the recovered version and the on-disk save — currently the autosave file would just sit there with no recovery path.
9. Intercept `glfwWindowShouldClose` when `ls_dirty` is set and present a "Save changes before exiting?" modal before tearing down — currently closing the window discards every unsaved edit silently.
10. Wire `theories/Csv.v` into a "File → Import CSV" menu item that reads a path from a native file dialog, parses rows/cols into the active sheet at the selected anchor (or A1), and pushes one undo entry — currently `Csv.v` is unwired and there is no CSV ingest at all.
11. Add a "File → Export CSV" menu item that writes the active sheet's non-empty cells to a path from a native file dialog, RFC 4180 quoted — currently the only export besides PDF is the bespoke text save format.
12. Add XLSX import: parse the .xlsx zip archive (`xl/workbook.xml`, `xl/worksheets/sheet*.xml`, `xl/sharedStrings.xml`) into the workbook, mapping XLSX cell types onto `Cell` constructors — currently there is no path to ingest data from any other spreadsheet program.
13. Add XLSX export: emit the matching `.xlsx` archive structure for the workbook, preserving cell text, formulas, formats, merges, and charts — currently the workbook is locked into the bespoke text format and cannot be opened in Excel / LibreOffice / Numbers.
14. Add ODS (OpenDocument Spreadsheet) export — same archive shape as XLSX but for the OASIS ODF target, so users on the LibreOffice side can be served first-class.
15. Acquire an advisory `flock` on the save path at startup so a second running instance refuses to open the same file — currently two concurrent rocqsheet processes editing the same file race on save and corrupt it.
16. Build a per-cell dependency graph in `loop_state` (`ls_deps : list (CellRef * list CellRef)`) populated incrementally from formula parse and torn down on commit — currently every render frame re-evaluates every visible cell from scratch with no notion of what has changed.
17. Memoize `eval_iter` results per cell in `ls_eval_cache : list (CellRef * EvalResult)` keyed by `(cell_index, generation_counter)`, invalidating the entry and every transitive dependent on commit — currently the runtime spends O(visible_cells × formula_depth) work per frame.
18. Cache per-chart computed values in `ls_chart_cache : list (Chart * list Z)`, invalidated when any of the chart's range cells dirty — currently every chart re-walks its underlying range every frame.
19. Replace `lookup_format`'s O(N) linear scan with a `PrimArray.array (option CellFormat)` indexed by `cell_index`, giving O(1) format lookup per cell render — currently a workbook with thousands of formatted cells stalls the render loop.
20. Replace the assoc-list `ls_other_sheets : list (int * Sheet)` with a `PrimArray.array Sheet` of length `NUM_SHEETS`, giving O(1) sheet lookup and removing the assoc search from the per-frame tab path — currently `lookup_sheet_or_new` walks the full assoc list per query.
21. Convert `do_clear` / `switch_to_sheet` / `do_undo` to mutate the workbook in place via `persistent_array`'s rvalue `set` overload instead of constructing new sheets from scratch — currently each invocation snapshots the entire `persistent_array<Cell>` of every sheet it touches.
22. Convert `apply_load_lines` from a per-line Coq Fixpoint that rebuilds `loop_state` at every step into a single-pass C++ helper that mutates a working `loop_state` via `std::move` and only emits the final state — currently loading a 50KB save file triggers thousands of full `loop_state` reconstructions.
23. Move file I/O (`file_read`, `file_write`, autosave, .bak rotation) onto a background thread fed by a single-producer single-consumer queue back to the GUI thread — currently a save of a large workbook blocks the main loop and the UI freezes.
24. Cap parser input length at 4096 chars at `parse_formula`'s entry — currently a pathological 1MB string in a cell pegs a CPU core indefinitely.
25. Promote `NUM_COLS` and `NUM_ROWS` to runtime fields on `loop_state` (`ls_num_cols`, `ls_num_rows`) and resize the underlying `persistent_array<Cell>` on grow operations — currently the grid is hardcoded at 260×200 at compile time and cannot grow with the workbook.
26. Promote sheet count to runtime: replace the fixed-`NUM_SHEETS` array with a `persistent_array<Sheet>` whose length comes from `ls_sheet_count`, and surface "Add Sheet" / "Remove Active Sheet" menu items — currently the workbook is hardwired to exactly 16 slots.
27. Add cross-sheet references: introduce `ECrossRef : int -> CellRef -> Expr`, extend the parser to accept `Sheet2!A1` syntax, and have `eval_iter` resolve via the right sheet from `ls_other_sheets` — currently formulas can only reference cells on the active sheet.
28. Add range syntax to the parser: `A1:C5` parses to a new `Range` value (rectangle of CellRefs) and `SUM` / `AVG` / `MIN` / `MAX` / `COUNT` accept a `Range` instead of two `CellRef`s — currently aggregations require the user to spell out the explicit corner cells, and an `A1:C5` literal in the formula bar is a parse error.
29. Wire `theories/NamedRanges.v` into the formula parser so a name lookup table replaces a name token with its underlying cell reference at parse time, and surface a "Define Named Range" modal — currently named ranges exist as a Coq definition with no path from the user-typed formula to the resolved ref.
30. Move `ls_charts` from `loop_state` onto each `Sheet` so charts are per-sheet — currently the chart panel always shows the workbook-wide list, regardless of which sheet is active.
31. Add `ls_validation_rules : list (CellRef * ValidationRule)` with rules like "integer between 0 and 100", "one of {Yes,No,Maybe}", and "matches regex", enforced at commit time and wired to a "Data → Validation" modal — currently any text can land in any cell, so spreadsheets cannot model constrained inputs.
32. Add a per-cell coercion table that promotes a `CStr "5"` to `CLit 5` (and `CDate "2026-01-01"` to its epoch-day `CLit`) at evaluation time when the consuming operator wants an integer or date — currently a numeric formula referencing a string-typed cell fails with `EErr`.
33. Add a `CDate : int -> Cell` constructor (epoch days) plus parser support for `YYYY-MM-DD` literals and a NumberFormat `NFDate` family — currently dates are stored as strings or integers with no semantic.
34. Ship `SUMIF`, `COUNTIF`, `AVERAGEIF` over a range, a predicate (`EGt`/`ELt`/`EEq` against a literal), and an aggregation column — currently conditional aggregates require the user to compose `EIf` per cell and sum the result, which is verbose and quadratic.
35. Ship `VLOOKUP`, `HLOOKUP`, `INDEX`, `MATCH` with proper exact / approximate match semantics — currently lookup must be hand-rolled out of `EIf` chains.
36. Ship `UPPER`, `LOWER`, `TRIM`, `REPLACE`, `SPLIT`, `FIND`, plus a `REGEX_MATCH` / `REGEX_REPLACE` pair backed by a Coq-extracted Thompson-NFA engine — currently the only string ops are `CONCAT` / `LEN` / `SUBSTR`.
37. Ship `STDEV`, `VAR`, `MEDIAN`, `MODE`, `RANK`, `PERCENTILE` over ranges — currently any statistical work requires a separate tool.
38. Ship `PMT`, `NPV`, `IRR`, `FV`, `PV` financial formulas with the standard Excel signatures and discount-rate handling — currently the workbook cannot model loans, investments, or DCFs.
39. Ship `TODAY`, `NOW`, `DATEDIF`, `WEEKDAY`, `EDATE`, `EOMONTH` operating on the new `CDate` type — currently date math is not expressible.
40. Ship `IFS` (multi-conditional) and `SWITCH` (value-matched dispatch) — currently the only conditional is binary `EIf`, requiring nested chains for n-way branching.
41. Ship `RAND` and `RANDBETWEEN` driven by a deterministic PRNG seeded from a new `ls_seed` field, so saved workbooks reproduce their random values bit-identically across reloads — currently there is no source of randomness.
42. Ship array / spill formulas: an `EArrayFormula : Range -> Expr -> Expr` that evaluates the inner expression with each cell of the range as the implicit input and writes the results into a destination range — currently every cell of a tabular computation must be authored individually.
43. Ship matrix ops `MMULT`, `TRANSPOSE`, `MINVERSE`, `MDETERM` over ranges — currently linear-algebra workloads cannot be expressed.
44. Saturate `EFAdd` / `EFSub` / `EFMul` / `EFDiv` to `+/-INF` on overflow rather than silently propagating IEEE NaN, with an explicit `EFNan` constructor for genuine NaN-producing math — currently float overflow corrupts downstream cells with NaN that compares unequal to itself.
45. Add explicit type-coercion operators `TO_INT`, `TO_FLOAT`, `TO_STR`, `TO_BOOL`, `TO_DATE` and use them at `eval_iter` boundaries when an operator's arity wants a different type than the operand provides — currently the only path between types is to bounce through the formula bar by hand.
46. Promote `EPow x (-n)` to an `EFVal (1/x^n)` result instead of `EErr` — currently negative integer exponents are rejected even though their reciprocals are well-defined.
47. Locale-aware number parsing: a `parse_int_literal` and `parse_float_literal` that consult `ls_locale` for thousands separator, decimal separator, and minus-sign convention — currently the parser only accepts a single contiguous run of digits.
48. Build a reusable ImGui modal-popup framework in `imgui_helpers::modal_begin` / `modal_end` plus a typed `ModalState` enum on `loop_state` — currently every "dialog" piggybacks on the formula bar, which is read-only-during-modal-context, and there is no way to surface a confirm prompt.
49. Add a real native file-open / file-save dialog by linking `nativefiledialog-extended` and exposing `imgui_helpers::file_dialog_open` / `file_dialog_save` axioms — currently `Save As` reads the path from the formula bar, which is unusable for anyone who hasn't read the menu's parenthetical hint.
